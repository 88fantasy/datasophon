# 单元测试覆盖率查看指南

## 覆盖率概览

CommonBtnList 组件的单元测试覆盖率：
- **语句覆盖率 (Statements)**: 100% ✅
- **分支覆盖率 (Branches)**: 100% ✅  
- **函数覆盖率 (Functions)**: 100% ✅
- **行覆盖率 (Lines)**: 100% ✅

这意味着组件的所有代码行、所有分支条件、所有函数都被测试覆盖。

## 查看覆盖率的方法

### 方法 1️⃣：使用命令行查看覆盖率摘要

```bash
pnpm test:coverage
```

这将在终端输出覆盖率统计表格。

### 方法 2️⃣：查看完整的 HTML 覆盖率报告

生成覆盖率后，会在 `coverage/` 目录下生成 HTML 报告：

1. **打开 HTML 报告**
   - 文件位置：`coverage/index.html`
   - 在浏览器中打开此文件即可看到详细的覆盖率报告
   - 支持按文件查看，点击文件名可查看具体的代码行覆盖情况

2. **查看组件的覆盖率详情**
   - 打开 `coverage/index.tsx.html` 查看 CommonBtnList 组件的详细覆盖情况
   - 绿色代码行表示已覆盖
   - 红色代码行表示未覆盖

### 方法 3️⃣：VS Code 中查看

可以在 VS Code 的文件浏览器中导航到 `coverage/` 文件夹，然后：
- 右键点击 `index.html` 
- 选择"使用实时服务器打开"或类似选项
- 或直接双击打开

## 覆盖率详细指标

| 指标 | 说明 | 当前值 |
|------|------|--------|
| **语句覆盖率** | 代码中每个语句至少被执行一次 | 100% |
| **分支覆盖率** | 每个条件分支（if/else 等）都被执行 | 100% |
| **函数覆盖率** | 所有定义的函数都被调用 | 100% |
| **行覆盖率** | 源代码的每一行都被执行 | 100% |

## 测试覆盖的场景

✅ **基础渲染** - 验证组件正确渲染按钮列表
✅ **按钮属性** - 验证 Ant Design 属性传递正确
✅ **点击处理** - 验证回调函数执行和异步操作
✅ **加载状态** - 验证加载状态管理和更新
✅ **多按钮管理** - 验证多个按钮的独立管理
✅ **边界情况** - 特殊字符、错误处理等
✅ **性能优化** - React.memo 和大列表处理

## 运行所有项目的测试覆盖率

如果想查看整个项目的测试覆盖率：

```bash
# 运行所有测试并生成覆盖率报告
pnpm test:coverage

# 查看详细的 HTML 报告
# 打开 coverage/index.html
```

## 常见命令

```bash
# 交互式开发模式（监听文件变化）
pnpm test

# 单次运行所有测试
pnpm test:run

# 生成覆盖率报告（推荐）
pnpm test:coverage

# 使用 UI 界面运行测试
pnpm test:ui

# 只运行特定文件的测试
pnpm test:run src/components/Common/CommonBtnList/index.test.tsx

# 只运行特定文件的覆盖率报告
pnpm test:coverage src/components/Common/CommonBtnList/index.test.tsx
```

## 丰富的覆盖率报告信息

HTML 报告包含：
- 📊 整体覆盖率统计
- 📁 按文件分类的覆盖率
- 🔍 代码级别的覆盖详情（每行是否已覆盖）
- 🎨 颜色编码（绿=已覆盖，红=未覆盖）
- 🔢 精确的覆盖数据

## 提高覆盖率的建议

如果未来覆盖率不足 100%，可以：

1. **识别未覆盖的代码** - 查看 HTML 报告中的红色代码行
2. **添加新的测试用例** - 针对未覆盖的分支编写测试
3. **检查边界情况** - 确保所有可能的代码路径都有测试
4. **使用覆盖率阈值** - 在 vitest.config.ts 中配置最小覆盖率要求

## 覆盖率报告位置

```
项目根目录/
├── coverage/
│   ├── index.html              ← 打开这个文件查看总覆盖率
│   ├── index.tsx.html          ← CommonBtnList 组件的覆盖率详情
│   ├── coverage-final.json     ← 机器可读的覆盖率数据
│   └── ...其他资源文件
└── ...
```

## 注意事项

- 覆盖率报告在 `coverage/` 目录中，每次运行 `pnpm test:coverage` 时会被重新生成
- 如果要保存报告，建议将 `coverage/` 目录备份或版本控制
- HTML 报告需要在浏览器中打开才能交互式查看
- 可以将 `coverage/` 目录添加到 `.gitignore` 中（通常不需要版本控制）
